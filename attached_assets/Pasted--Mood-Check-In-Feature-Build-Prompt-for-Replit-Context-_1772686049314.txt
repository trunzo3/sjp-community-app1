# Mood Check-In Feature — Build Prompt for Replit

## Context

This is an existing mobile-first React + TypeScript + Tailwind app built on Node/Express with PostgreSQL and Drizzle ORM. The app is a private community platform for Saint John's Program for Real Change. The current bottom nav has 5 tabs: Home, Community, Resources, Events, Profile. The Profile page (`/profile`) currently shows an editable bio, the user's post history, and links to admin or story features depending on role.

Full tech stack and schema details are in the PRD already loaded in this project.

## 3. Database: New Table `mood_checkins`

Add to `shared/schema.ts`:

```
mood_checkins table:
  id            UUID, primary key
  userId        UUID, FK → users.id, not null
  coreEmotion   Text, not null          (e.g. "peaceful", "hopeful", "heavy", "restless", "strong")
  midEmotion    Text, not null          (e.g. "calm", "content", "relieved", "resting")
  outerEmotion  Text, not null          (e.g. "centered", "still", "grounded", "at_ease")
  coreColor     Text, not null          (hex color of the core emotion)
  midColor      Text, not null          (hex color of the mid emotion)
  outerColor    Text, not null          (hex color of the outer emotion)
  outerLabel    Text, not null          (display label, e.g. "Centered")
  journalEntry  Text, nullable          (optional private journal text)
  checkedInAt   Timestamp, not null, default now()
```

Add a unique constraint on `(userId, date(checkedInAt))` — one check-in per user per calendar day. If a user checks in again the same day, update the existing record (upsert).

**Data retention:** Store indefinitely. The data is tiny and the longitudinal record is valuable. The frontend controls how much to display.

## 4. API Endpoints

All endpoints require `requireAuth` middleware. Mood data is strictly private — users can only access their own records.

```
GET    /api/mood/today          → Returns today's check-in for the logged-in user, or null
POST   /api/mood                → Create or update today's check-in
         Body: { coreEmotion, midEmotion, outerEmotion, coreColor, midColor, outerColor, outerLabel, journalEntry? }
         Returns: the created/updated record
GET    /api/mood/history        → Returns the logged-in user's check-ins for the last 90 days
         Returns: Array of check-in records, ordered by date descending
```

**Privacy enforcement:** These endpoints must ONLY return data for the authenticated user. No staff or admin override. Mood data is never visible to anyone else, period. Do not add mood data to any admin panel or reporting view.

## 5. The Mood Check-In Flow

This is a 3-step progressive drill-down. The user starts broad and gets specific. Each step is its own screen state with a crossfade transition between steps.

### Step 0: Core Emotions
Display 5 large tappable cards, one per core emotion. Each card shows:
- A filled circle in the core emotion's color (prominent, 48-52px)
- The emotion label in that color
- A subtitle line showing the 4 mid-level words joined by " · "

The 5 core emotions and their colors:
- **Peaceful** — `#3B82F6` (blue)
- **Hopeful** — `#FACC15` (yellow)
- **Heavy** — `#8B5CF6` (violet)
- **Restless** — `#EF4444` (red)
- **Strong** — `#22C55E` (green)

### Step 1: Mid-Level Emotions
Display a 2x2 grid of cards. Each card shows:
- A smaller color circle in that mid-emotion's unique shade
- The mid-emotion label in its color
- A subtitle showing the 4 outer words

**Each mid-level option has its own distinct shade within the parent hue family.** This is critical — they should NOT all be the same color. See the full color taxonomy below.

A breadcrumb chip at the top shows the selected core emotion in its color.

### Step 2: Outer / Precise Emotions
Display a single-column list of 4 options. Each shows:
- A thick left border in that word's unique color
- A small color dot (16px)
- The word label

**Each outer word has its own unique shade**, slightly different from its siblings. See taxonomy below.

Breadcrumb chips show Core → Mid with their respective colors.

### Step 3: Journal + Save
Display:
- A summary card with the outer emotion's specific color as the full background. White text (or dark text on yellow). Shows "Today you're feeling [word]" and a 3-dot color trail showing Core → Mid → Outer colors.
- An optional journal prompt: "Want to say more? Tap to write…" as a dashed-border ghost button. On tap, expands to a textarea. Placeholder: "Whatever's on your mind. This is just for you."
- A save button in the outer emotion's color
- Copy below the button: "No journal needed — just tap Save"

After save, show a confirmation with the color circle, the word, the 3-dot trail, journal text if entered, and "This is yours. Nobody else sees it." Offer two buttons: "New Check-In" and "View My Colors".

### Global UI Elements

**Back button** in the header on steps 1-3, colored to match the active emotion.

**"Private · Only You"** badge in the top right corner on every step.

**Progress dots** — 4 dots below the header. Filled dots use the active color. Step 0 dots can use a rainbow gradient.

## 7. My Colors — 90-Day History View

This section appears on the Me page below the check-in card, and is also accessible from the post-save confirmation screen.

### Color Strip
A horizontal strip showing the last 90 days. Each day is a small colored cell using the outerColor of that day's check-in. Days with no check-in show as a neutral light gray (`#E5E1DE`). Most recent day is on the right. The strip should be in a scrollable container if needed, or rendered as a calendar grid (7 columns for days of the week, ~13 rows). The calendar grid is preferred because it shows weekly patterns and is more compact.

### Calendar Grid Layout
- 7 columns (Sun–Sat), labeled at top
- Each cell is a small rounded square (~36px)
- Filled with the outerColor if a check-in exists, `#E5E1DE` if not
- Tapping a colored cell shows a small tooltip or inline detail: the date, the emotion label, and the 3-dot color trail
- Current day has a subtle border or ring to indicate "today"
- Scroll vertically if more than ~5 weeks are visible

### Timeline List (below the grid)
A reverse-chronological list of recent check-ins (last 14 days). Each entry shows:
- A 3-dot vertical color trail (Core → Mid → Outer)
- The outer emotion label in its color
- The date
- A color swatch (rounded square) in the outer color

This list should be collapsible or "show more" to avoid making the page too long.

## 8. Text Contrast Helper

Several emotion colors are light (especially yellows and light greens/blues). Use this logic for text-on-color decisions:

```typescript
function textOnColor(hex: string): string {
  const r = parseInt(hex.slice(1, 3), 16);
  const g = parseInt(hex.slice(3, 5), 16);
  const b = parseInt(hex.slice(5, 7), 16);
  return (r * 0.299 + g * 0.587 + b * 0.114) > 160 ? "#302D2E" : "#FFFFFF";
}
```

Use this everywhere a color is used as a background for text — summary cards, breadcrumb chips, save buttons, calendar cells with labels, etc.

## 10. Design Constraints

- **Mobile-first.** The Me page, the check-in flow, and the history view must all work perfectly at 375px width.
- **Typography:** Use the existing app font stack. Use `DM Serif Display` for emotion labels and headings within the mood feature if available, otherwise fall back to the app's heading font.
- **The check-in flow should feel distinct** from the rest of the app — warmer, more personal, like a moment of pause. The vivid emotion colors are the primary design element. Cards should have subtle shadows and smooth transitions.
- **Privacy is non-negotiable.** No mood data in the community feed, no mood data visible to staff or admin, no mood data in any admin panel, no mood data in any API endpoint that returns data about other users. This is strictly personal.
- **Animations:** Crossfade between steps (opacity + translateY, ~250ms). Hover/tap states on cards should scale slightly and deepen the border color. The summary card on step 3 should have a prominent box-shadow in the emotion's color.
