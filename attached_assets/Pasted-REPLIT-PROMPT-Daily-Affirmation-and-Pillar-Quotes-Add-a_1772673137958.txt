REPLIT PROMPT: Daily Affirmation and Pillar Quotes
Add a daily affirmation card to the home screen that rotates through a curated set of short quotes and reflections, each tied to one of the five Saint John's Foundations for Real Change pillars: Community, Confidence, Resilience, Readiness, and Wellness.

WHERE IT LIVES
Place the affirmation card on the home screen directly below the welcome banner and above the Stories of Change carousel. It should be present for all user roles: clients, alumni, staff, and admin.

HOW IT WORKS
One affirmation displays per day, the same one for all users on that day
Rotation is date-based: use the current date to deterministically select from the pool, so the quote is consistent for everyone throughout the day and changes at midnight
The card shows the quote, the pillar it belongs to, and the pillar's icon
Include a small "pillar label" beneath the quote in muted text, for example "Resilience" with its icon, so users gradually learn the framework through daily exposure
No user interaction required. No likes, no saves, no sharing. It is a moment, not a feature.
The card should feel warm and grounded, not like a motivational poster. Gentle typography, soft background tint using the pillar's existing color from the design system

PILLAR COLOR AND ICON MAPPING
Use the existing design system colors already defined for each pillar:
Community — Teal #34737A, icon: Users Confidence — Peach #EEBBA7, icon: Sunrise or Award Resilience — Sage green #5DA592, icon: Leaf or TrendingUp Readiness — Dusty lavender #979DB6, icon: CheckCircle or Compass Wellness — Amber #C8882A, icon: Heart or Sun
All icons from lucide-react, already in the stack.

SCHEMA AND DATA STORAGE
No database table needed. Store the quote pool as a static JSON array in the codebase, for example in client/src/data/affirmations.ts. Each entry should include:
{
  pillar: "resilience",
  quote: "...",
  attribution: null
}
Attribution is null for now. If Saint John's wants to attribute quotes to staff, alumni, or external figures later, the field is ready.
The daily rotation logic lives on the frontend. Use new Date().toDateString() hashed or converted to a numeric index modulo the total quote count to select the day's quote. No API call needed.

DO NOT CHANGE
The welcome banner above the card
The Stories of Change carousel below it
Any other home screen element
The pillar filtering system in the Resources section, which uses the same pillar names and should remain independent of this feature

ACCEPTANCE CRITERIA
Affirmation card appears on the home screen for all users
Quote changes each day and is consistent for all users on the same day
Correct pillar color, tint, and icon display for each quote
Pillar name is visible but secondary to the quote text
Card is readable and well-proportioned on a 390px screen
Static data file is clearly commented as pending SJP content review
