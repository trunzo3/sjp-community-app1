## Check 1: Database

The user_progress table should have six category values: journey, employment, housing, finance, parenting, community. Each stores an integer from 0 to the category's max — NOT 0-100 percentages. Verify the enum or column values are correct. If the old five pillars (community, confidence, resilience, readiness, wellness) are still in the schema, migrate to the new six.

## Check 2: Slider Max Values

Each category has a specific maximum number of sessions:
- Journey: 104
- Employment: 11
- Housing: 8
- Finance: 8
- Parenting: 12
- Community: 4

In the Admin Panel > Users progress sliders AND in the client self-edit view, each slider's max must match these numbers. The slider should step in increments of 1 (whole numbers only). Verify sliders are not still using 0-100.

## Check 3: Overall Percentage Calculation

The overall percentage in the large My Journey ring on the Home page = (sum of all six current values) / 147 × 100, rounded to the nearest whole number. 147 is the total of all six maximums. Verify this calculation is correct and not still averaging five percentages.

## Check 4: Client Self-Edit

Save button commits changes. Cancel returns to Home page. No role dropdown, no stage dropdown, no graduation date in this view. If this feature does not exist yet, build it.
