# SJP Community Platform — Prompt 1 of 2

**Create a named checkpoint called "pre-build" before running this prompt.**

---

Build a private, mobile-first community web app called "SJP Community" for Saint John's Program for Real Change, a Sacramento nonprofit serving women experiencing homelessness, addiction, justice involvement, and domestic violence. This is a closed community platform — not a case management tool, not a clinical dashboard, not a public website. Authenticated users only.

Tech stack: React frontend, Node.js/Express backend, PostgreSQL database. Mobile-first at 430px max-width, responsive up. Deploy to a live URL accessible on real phones.

This is Prompt 1 of 2. This prompt builds the foundation, all five user-facing pages with full features, and seed data. Prompt 2 will add the Admin Panel, guided storytelling flow, and alumni survey system.

---

## 1. DATABASE SCHEMA

Create a PostgreSQL database with these exact tables and fields. Do not rename fields or change types.

**users**
- id (uuid, primary key)
- email (text, unique, not null)
- password (text, hashed, not null)
- first_name (text, not null)
- last_name (text, not null)
- role (enum: 'client', 'alumni', 'staff', 'admin')
- stage (enum: 'client', 'alumni')
- graduation_date (date, nullable — must exist from day one, nullable, no default)
- bio (text, nullable)
- avatar_color (text — hex color string for initial-based avatar circle)
- created_at (timestamp, default now)

**posts**
- id (uuid, primary key)
- author_id (uuid, foreign key → users.id)
- content (text, not null)
- post_type (enum: 'update', 'win', 'question', 'need', 'milestone')
- pinned (boolean, default false)
- created_at (timestamp, default now)

**replies**
- id (uuid, primary key)
- post_id (uuid, foreign key → posts.id)
- author_id (uuid, foreign key → users.id)
- content (text, not null)
- created_at (timestamp, default now)

**resources**
- id (uuid, primary key)
- name (text, not null)
- description (text)
- pillar (enum: 'community', 'confidence', 'resilience', 'readiness', 'wellness')
- type (enum: 'partner', 'program', 'service', 'opportunity')
- provider_name (text)
- phone (text, nullable)
- website_url (text, nullable)
- applicable_stages (text array — e.g. ['client', 'alumni'] or ['client'] or ['alumni'])
- created_by (uuid, foreign key → users.id)
- created_at (timestamp, default now)

**events**
- id (uuid, primary key)
- name (text, not null)
- event_type (enum: 'community_meeting', 'workshop', 'celebration', 'class', 'partner_session')
- date (date, not null)
- start_time (time, not null)
- end_time (time, not null)
- location (text)
- description (text)
- applicable_stages (text array)
- created_by (uuid, foreign key → users.id)
- created_at (timestamp, default now)

**stories**
- id (uuid, primary key)
- author_id (uuid, foreign key → users.id)
- step1_content (text — "Where were you?")
- step2_content (text — "What changed?")
- step3_content (text — "Where are you now?")
- share_externally (boolean, default false)
- approval_status (enum: 'pending', 'approved', 'community_only')
- featured (boolean, default false)
- created_at (timestamp, default now)

**surveys**
- id (uuid, primary key)
- user_id (uuid, foreign key → users.id)
- interval_months (integer — value is 3, 6, or 12)
- still_employed (boolean)
- job_title (text, nullable)
- raise_or_promotion (boolean)
- promotion_details (text, nullable)
- housing_status (enum: 'stable', 'transitional', 'unstable')
- support_needs (text, nullable)
- submitted_at (timestamp, default now)

## 3. CONTENT VISIBILITY RULES

Enforce on the backend, not just the frontend.

- Users with role `client` or `alumni` see resources and events where their `stage` value is in the item's `applicable_stages` array, OR where the item is tagged for all stages (both 'client' and 'alumni' in the array).
- Users with role `staff` or `admin` see ALL resources and events regardless of stage tags.
- Stage badge labels on resource and event cards are visible ONLY to staff and admin.
- "+ Add" buttons on Resources and Events pages are visible ONLY to staff and admin.
- "Admin Panel" link on Profile page is visible ONLY to staff and admin.
- "Share Your Story" on Home page and Profile page is visible ONLY to users with role `alumni`.
- Survey prompts appear ONLY for alumni with non-null `graduation_date` at correct intervals.

## 12. TONE AND COPY RULES

This app serves women experiencing homelessness, addiction, justice involvement, and domestic violence. Every word matters.

**Do:**
- First names only in all community-facing views
- Strength-based framing: "Share your story" not "Submit your testimony"
- Warm, direct language: "Your journey can inspire someone who's just starting"
- Celebrate milestones visually

**Do not:**
- Use clinical language in the UI: no "clients," "cases," "interventions," "treatment plans"
- Use surveillance language: no "you are required to," "your activity is monitored"
- Use deficit framing: no "at-risk," "struggling," "in crisis"
- Use generic placeholder text — all seed data should feel real and specific to this community

## 13. THE FIVE FOUNDATIONS FOR REAL CHANGE

These are SJP's five pillars. They organize the resource library and represent the graduate profile framework:

1. **Community** — Cultivating strong, supportive relationships and meaningful connections. Building healthy relationships and support networks.
2. **Confidence** — Demonstrating inner strength and a positive, proactive mindset. Nurturing self-worth and self-esteem with an "I can" attitude.
3. **Resilience** — Confident self-advocacy and active agency. Standing up for yourself and your family. Overcoming challenges with strength and determination.
4. **Readiness** — Prepared with essential tools, knowledge, and resources for financial independence and stability in employment and housing.
